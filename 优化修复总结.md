# iOS签名工具优化修复总结

## 问题分析与解决方案

### 1. ❌ dylib注入没效果问题
**问题原因**: 
- 传统dylib注入只是传递给了zsign，但zsign已忽略dylib参数
- 没有实际调用MachO注入函数

**解决方案**:
- ✅ 在签名前添加传统dylib注入逻辑
- ✅ 复制dylib文件到Frameworks目录
- ✅ 使用纯Python MachO注入工具实际注入动态库
- ✅ 添加完整的验证机制

**关键代码修改**:
```python
# src/core/sign_engine.py
if inject_dylibs:
    # 复制dylib到Frameworks目录
    # 生成install name格式
    # 调用inject_frameworks_to_app实际注入
    # 验证注入结果
```

### 2. ❌ 解压依赖本地7z问题
**问题原因**:
- 依赖本地安装的7z工具
- 有PowerShell和zipfile备选方案，增加复杂性

**解决方案**:
- ✅ 只使用内置7z工具(`tools/7Z/7Z.exe`)
- ✅ 删除PowerShell和zipfile备选方案
- ✅ 简化解压逻辑，提高可靠性
- ✅ 确保路径处理正确

**关键代码修改**:
```python
# src/utils/file_utils.py
# 使用内置7z工具解压
tools_dir = os.path.join(base_dir, "tools", "7Z")
seven_zip_path = os.path.join(tools_dir, "7Z.exe")
```

### 3. ❌ zsign路径参数问题
**问题原因**:
- 中文路径导致zsign参数截断
- "Invalid path! D:\签名\iossign\sign\dist\iOS签名工具\9"

**解决方案**:
- ✅ 所有路径参数使用引号包围
- ✅ 在Windows上使用shell=True处理引号路径
- ✅ 确保参数正确传递给zsign

**关键代码修改**:
```python
# src/utils/zsign_utils.py
"-k", f'"{p12}"',  # 使用引号包围路径
"-m", f'"{prov}"',  # 使用引号包围路径
args.append(f'"{app}"')  # 使用引号包围路径

# 在Windows上使用shell=True
use_shell = platform.system() == "Windows"
process = subprocess.Popen(
    args if not use_shell else ' '.join(args),
    shell=use_shell,
    ...
)
```

### 4. ⚠️ zsign参数规范性
**当前参数**: `-k -p -m -b -n -r -f -e -z 9`

**参数说明**:
- `-k`: 证书路径 ✅
- `-p`: 证书密码 ✅  
- `-m`: 描述文件路径 ✅
- `-b`: Bundle ID ✅
- `-n`: 应用名称 ✅
- `-r`: 版本号 ✅
- `-f`: 强制签名 ✅
- `-e`: 深度签名/entitlements ✅
- `-s`: 跳过验证（仅在重试时使用）✅
- `-z 9`: SHA256签名算法 ✅

**结论**: 参数使用正确，符合zsign标准
